## Narrative Arc & Shot Map

### Act A — Setup (0:00–0:20)

### Act B — Test Capital (0:20–2:30)
- Tests = walls of the mold, constraints the generated code cannot cross
- CodeRabbit: AI code 1.7x more issues, 75% more logic errors, 8x perf problems
- DORA: AI without tests = instability; AI with tests = delivery amplification
- Bug fix workflow: add a wall, not a patch → permanent, prevents forever
- Ratchet effect: tests only accumulate, mold only gets more precise
- Z3 SMT solver: same SAT/SMT technology as Synopsys chip verification
- Mathematical proof vs sampling — "proven, not just tested"
- Visuals: wall-building animation, ratchet mechanism, Z3 proof visualization

### Act C — Prompt Capital (2:30–3:45)
- Prompt = specification of what and why (not how)
- Implementation varies; behavior is locked
- 1/5 to 1/10 the size of generated code
- Context window: prompts are NL, models trained on 30x more NL
- NL comments improved code generation 41%
- Author-defined context vs machine-assembled context
- Visuals: prompt document with radiating code, size comparison, context diagram

### Act D — Grounding Capital (3:45–4:30)
- Grounding = learned from past generations
- Style, patterns, team conventions captured automatically
- Feeds forward into future generations
- Visuals: learning loop animation, style consistency examples

### Act E — Synthesis (4:30–5:30)
- Prompt + tests + grounding = intent + constraints + style
- Complete specification
- "The code is output. The mold is what matters."
- Visuals: three components merge into unified mold diagram
