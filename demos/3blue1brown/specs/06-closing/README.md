## Key Visual Elements

### Glow Rule (Critical)
- Prompts, tests, and grounding ALWAYS glow -- they are the valuable artifacts
- Generated code NEVER glows -- it is "just output," "just plastic"
- This rule is the visual thesis of the entire video and must be strictly maintained

### Color Palette (Part 6)

- **Prompt (Blue):** #4A90D9 - cool specification color
- **Tests/Walls (Amber):** #D9944A - warm constraint color
- **Grounding (Green):** #5AAA6E - style/consistency color
- **Generated Code:** Neutral gray (#A0A0A0), no glow, no emphasis
- **Code Background:** Dark (#1E1E2E)
- **Canvas Background:** Dark (#1a1a2e)
- **Success:** Green (#5AAA6E)
