# Identified Issues - Step 11

## 1. Interface/Requirement Inconsistencies

### `pdd/prompts/preprocess_python.prompt`
- **Issue**: The `<pdd-interface>` declares the parameter `exclude: list[str] | None = None`, but **Requirement 1** and the **Curly Brace Handling** section use `exclude=None`.
- **Impact**: This will cause signature drift errors or architecture conformance failures when generating `pdd/preprocess.py`.
- **Context**: `preprocess_main_python.prompt` correctly passes `exclude=exclude`.

## 2. Missing Flag Propagation

### AST-based Compression (`--compress`)
- **Issue**: The `compress` flag is supported in `sync_main` and `sync_orchestration`, but it is **missing** from `code_generator_main` and `code_generator`.
- **Impact**: Even when `pdd sync --compress` is used, the actual code generation phase (which calls `preprocess` via `code_generator`) will not apply compression to includes unless they explicitly specify `mode="compressed"`.
- **Resolution**: Add `compress: bool = False` to the signatures of `code_generator_main` and `code_generator`, and ensure it is passed through to the `preprocess()` call.

### Evidence Manifest (`--evidence`)
- **Issue**: The `evidence` flag is documented in `README.md` and supported by `evidence_manifest_python.prompt`, but it is **missing** from the signatures of `sync_main` and `sync_orchestration`.
- **Impact**: The `pdd sync --evidence` command will fail to parse or ignore the flag, and the evidence manifest will not be written.
- **Resolution**: Add `evidence: bool = False` to `sync_main` and `sync_orchestration`, and implement the call to `write_evidence_manifest` (using `validation_from_sync`) in the sync workflow.

## 3. `architecture.json` Inaccuracies

- **Issue**: Several `filepath` entries for prompts are missing the `pdd/` prefix (e.g., `prompts/agentic_bug_step4_reproduce_LLM.prompt` instead of `pdd/prompts/...`).
- **Issue**: Some entries point to non-existent code files (e.g., `extensions/recruiting/resurface_check.py`).
- **Impact**: `ORCHESTRATOR_POSTCHECK_WARNINGS` are triggered, and automated tools may fail to find the files.

## 4. Documentation Sync Silent Drops

- **Issue**: Step 10 reported `DOC_SYNC_SILENT_DROPS` in `README.md` and `docs/whitepaper.md`.
- **Finding**: Fragments like `` ` graph. `` suggest that Mermaid graph synchronization might be failing or that some sections are being incorrectly identified as redundant.

## 5. Circular Import Risks

- **Issue**: `include_query_extractor_python.prompt` correctly forbids top-level imports of `preprocess` to avoid circular dependencies (Requirement 12), but `preprocess_python.prompt` **lacks a corresponding requirement** to avoid top-level import of `IncludeQueryExtractor`.
- **Impact**: A circular import error will occur if `preprocess.py` imports `IncludeQueryExtractor` at module load time.

## 6. Prompt Cleanup

- **`pdd/prompts/preprocess_python.prompt`**: Requirement 6 contains a minor typo/instructional note: "(or `exclude` if that's the preferred name)". This should be removed.
- **`pdd/prompts/evidence_manifest_python.prompt`**: The `write_evidence_manifest` signature in `<pdd-interface>` is `(...)`. It should be more specific to guide the generator.
