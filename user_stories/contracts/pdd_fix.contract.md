# Contract: pdd fix

> Generated from the human-verified user story + issue. Do not hand-edit:
> it is regenerated to align whenever the Story changes. Humans verify the
> Story (`../story__pdd_fix.md`), not this contract.

## Covers

- R1: GitHub issue URLs route to agentic E2E fix.
- R2: Story files route to user-story-driven prompt fixes.
- R3: Manual inputs route to iterative code/test repair.
- R4: Mode-specific validation prevents ambiguous routing.

## Context

The `pdd fix` command is implemented by `prompts/commands/fix_python.prompt`.
It offers agentic issue mode, user story mode, and manual mode from one CLI
entrypoint.

## Acceptance Criteria

1. Given a GitHub issue URL and no manual override, when the user runs `pdd fix <issue-url>`, then the command invokes the agentic E2E fix workflow and forwards resume, clean-restart, force, CI, GitHub-state, timeout, cycle, verbosity, and test-protection choices.
2. Given exactly one `story__*.md` file and no manual override, when the user runs `pdd fix <story-file>`, then the command invokes user-story fix mode with the current Click context and global command settings.
3. Given manual inputs or `--manual`, when the user runs `pdd fix`, then the command invokes manual fix mode and iterates over the selected unit-test files.
4. Given `--clean-restart` without a valid agentic issue URL, when the user runs `pdd fix`, then the command rejects the request before running a workflow.
5. Given a workflow result, when the command completes, then it reports success/failure consistently and exits non-zero on failed repair.

## Oracle

These details matter for pass/fail:
- routing order: issue URL before story file before manual fallback
- story filename shape: basename starts with `story__` and suffix is Markdown
- forwarded agentic options and manual repair options
- changed-file and message shape in returned results
- non-zero exit on failed repair

## Non-Oracle

These details should not matter:
- exact console colors
- exact internal helper/module names beyond the public workflow boundary
- exact wording of non-user-facing errors
- downstream patch content generated by the fixer

## Negative Cases

- A story file must not be treated as a manual prompt/code path.
- `--clean-restart` must not be accepted for manual or non-issue URL invocations.
- Manual loop mode must not treat the error file as a unit-test file.
- The command must not swallow Click usage errors as successful handled errors.

## Non-Goals

- This story does not verify the internals of `run_agentic_e2e_fix`.
- This story does not prescribe a particular fixing strategy or model.
- This story does not cover `pdd bug` or `pdd test`.

## Candidate Prompts

- `prompts/commands/fix_python.prompt` — primary owner of the `pdd fix` CLI.
- `prompts/agentic_e2e_fix_python.prompt` — related agentic fix workflow.
- `prompts/fix_main_python.prompt` — related manual repair workflow.
- `prompts/user_story_tests_python.prompt` — related story-driven fix support.
